# Blend UI Rebuild - Feature-Parity Specification

The Blend UI (WebView2, visual spec `blend.html`) must reproduce 100% of the WinForms
functionality below. Derived from a full code inventory 2026-07-03.

## Architecture
- DI (Program.cs): all services singletons except `ValheimServer`, `IRconClient`,
  `MainWindow`, `PlayerDetailsForm`, `WorldPreferencesForm`, `BlendWindow` (transient).
- Startup: Program → SplashForm → CreateNewMainWindow(profile, autoStart). SplashForm
  supports MULTIPLE MainWindows (one per profile) - Blend must handle multi-profile.
- `--blend` arg (dev): runs BlendWindow directly, bypassing SplashForm (no orphan
  adoption / autostart).
- Message protocol (BlendWindow.cs): JS→C# `{id?, method, params}`; C#→JS reply
  `{id, ok, result|error}`; C#→JS push `{event, data}`. `win.*` = fire-and-forget
  (minimize/maximize/close/dragStart/resizeStart{edge}).

## Menus (File / Help)
- New Window → FormProvider.CreateNewMainWindow
- New Profile / Save Profile / Save Profile As / Load Profile (dropdown) /
  Remove Profile (dropdown) → ServerPrefsProvider
- Preferences → PreferencesForm | Set Directories → DirectoriesForm
- Open Settings Directory → ShellExecute(AppDataPath)
- Close → OnFormClosing (confirm if server running)
- Help: Manual URL, Port Forwarding URL, Bug Report form, Check for Updates
  (SoftwareUpdateProvider), About (version + build date + GitHub/Valheim links)

## Sidebar (7 nav items) + theme toggle (dark default, ThemeManager)
Server / Advanced / Details / Players / Mods / Config / Logs

## Tab: Server Controls
- ServerNameField, Start/Stop/Restart buttons (Restart = RestartWithCountdown([60,30,10])
  or CancelCountdown when countdown active)
- World select: radio new (freetext) vs existing (dropdown from disk), Settings →
  WorldPreferencesForm, Refresh (rescan disk), Open Folder
- Password field + show toggle + copy button
- CommunityServerField (public), ServerCrossplayField
- ServerPortField (default 2456)

## Tab: Advanced
- Save interval (600s), backups count (4), backup short (600s) / long (43200s)
- WriteServerLogsToFile, AutoStart profile, AutoRestart on crash (+delay 10s),
  EmptyServerRestart (+delay 5m), ScheduledRestart (+interval 6h),
  RconEnabled (+port 25575 +password) - checkboxes enable/disable their delay fields
- Directories: ServerExePath (change → UpdateModCapabilities), SaveDataFolderPath,
  open-folder buttons
- AdditionalArgs passthrough

## Tab: Details (read-only, refreshed ~500ms while visible)
- Local (127.0.0.1) / Internal LAN / External IP (IpAddressProvider async + events),
  copy buttons, port display
- Crossplay invite code (InviteCodeReady event) + copy
- Session uptime, last world save (+ms), rolling average save time (WorldSaved event)

## Tab: Players
- Table: name (GetPlayerDisplayName), status (Online/Joining/Leaving/Offline),
  updated (TimeAgo). Source PlayerDataRepository (log-parsed), EntityUpdated events,
  500ms refresh while visible.
- Announce box → Server.BroadcastNow (RCON `broadcast center {msg}`)
- Details button (PlayerDetailsForm), Remove offline player button
- Right-click menu: [Name] · Spawn X at (ItemCatalog + SpawnItemForm →
  Server.SpawnAtPlayerAsync: RCON `playerlist` → parse coords → `baka_spawn {prefab}
  {x,z,y} {amount} {level}`) · Promote/Remove admin, Permit/Unpermit
  (PlayerListService, adminlist/permittedlist) · Heal (`dmg {t} -1000000` VERIFY-LIVE) ·
  Smite (`dmg {t} 1000000` VERIFY-LIVE, confirm) · Teleport (`tp {t} {dest}`
  VERIFY-LIVE, prompt) · Kick (`kick {t}`, confirm) · Ban/Unban (bannedlist.txt +
  kick if online, confirm) · Copy ID · View Details
- Capability gating (RequiredModChecker): RCON mod missing → kick/heal/smite/tp hidden;
  devcommands missing → Spawn hidden. Re-check on ServerExePath change.

## Tab: Mods
- Table: FullName, InstalledVersion, LatestVersion ("-" if unknown), status
  ("Update!"/"Up to date")
- Scan (ModScanner.ScanPlugins + concurrent ThunderstoreClient.GetLatestAsync),
  busy-guards `_modScanInProgress/_modUpdateInProgress`, 30-min auto-scan timer
- Update All: confirm → if running: broadcast + 3s delay + Stop + wait ≤60s →
  ModUpdateService.UpdateModsAsync → restart if was running
- Auto-update checkbox → UserPreferences.AutoUpdateMods
- Right-click: Open mod folder · Remove mod (± config) - confirm w/ file list,
  backup to BepInEx\.bakaloader-removed, stop/restart server if running
  (ModRemovalService)
- DONE (Task 4): companion plugins (BakaLoaderKillAll, BakaLoaderSpawnHelper,
  BakaLoaderItemIndexer, MMHOOK) are hidden from the mods list (ModScanner
  CompanionFolderNames, exact folder-name match) in BOTH UIs, and ModRemovalService
  refuses to remove them. KillAll v1.2.0 is now bundled (Resources/KillAll/) and
  auto-installed by KillAllInstaller on server start, same pattern as SpawnHelper.
  MMHOOK is hide-only: it is generated by the ValheimModding-HookGenPatcher BepInEx
  patcher on each game launch (game-version-specific), so bundling would be wrong.
  Live-verified 2026-07-04: 69 plugin folders on disk → mods.scan returns 66.

## Tab: Config Editor
- List BepInEx/config/*.cfg (from ServerExePath) → select loads text → Save writes
  back (+confirm) → Reload rescans. Refresh on tab show.

## Tab: Logs
- View select: Server / Application. LogViewer streams; ApplicationLogger.LogBuffer
  catchup on startup. Clear, Save-to-.txt (SaveFileDialog), open logs folder.

## Tray icon
- Left-click → refocus. Menu: [Profile+status] / Start / Restart / Stop / Close
  (enabled by Server.Status). Minimize → hide from taskbar (tray only).

## Window lifecycle
- OnShown: orphan adoption check, IP init, player/mod data init, startup prefs
  (autostart), path warnings, tray, version log
- OnFormClosing: server running → confirm stop-and-close (wait for stop);
  Windows shutdown → stop w/o waiting
- OnFormClosed: ForceKillTrackedProcess safety net (Job Object)

## Dialogs to reproduce
- PreferencesForm: SaveProfileOnStart, CheckForUpdates, StartWithWindows
  (StartupHelper), StartMinimized, WriteLogFileField, PasswordValidation, DarkMode,
  DiscordWebhookUrl + ThreadId; OK/Cancel/Defaults
- DirectoriesForm: exe path + save data folder (validated), open buttons, defaults
- AboutForm: version, build date, GitHub/Valheim links
- PlayerDetailsForm: name (+edit), Steam ID (+copy), characters add/edit/remove,
  status display, refresh, dirty-save prompt on close
- SpawnItemForm: live search (top 500), label+category list, amount 1–9999
  (warn ≥50), level (creatures 0–2) / quality (equipment 0–5), double-click OK
- TextPromptPopout: generic one-field modal
- WorldPreferencesForm: preset vs custom modifiers + world keys
- SplashForm: profile picker + startup task queue + multi-window orchestration
- BugReportForm

## ValheimServer surface (events the Blend bridge must forward)
StatusChanged(ServerStatus Stopped/Starting/Running/Stopping), WorldSaved(ms),
InviteCodeReady(code), ServerCrashed, CountdownTick(string|null).
Hooks: GetPendingModUpdateCount, ApplyModUpdates (empty-server mod updates, 10m).
Auto-restart triads: crash (silent, delay), empty (5m, cancel-on-join), scheduled
(6h, countdown broadcasts 1h/30m/10m/5m/4m/3m/2m/1m).

## Timers
ServerRefreshTimer ~500ms (players/details when visible) · UpdateCheckTimer (~24h) ·
ModScanTimer (30m)
